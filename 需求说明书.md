# 需求说明书

## 1. 系统背景与目标
- 本项目为桌面端 Qt "欢乐斗地主" 单机游戏，包含发牌、抢地主、出牌、计分、动画与音效的完整流程，入口在 `main.cpp`，启动时先显示加载界面再进入主窗口。

## 2. 角色与参与者
- 固定 1 名用户与 2 名机器人，初始化时建立前后指针形成出牌顺序，性别随机。

## 3. 功能列表
- 启动/加载：无边框透明进度条，完成后进入主界面。
- 发牌：循环发牌并生成三张地主牌面板，附带音效与移动动画。
- 抢地主/叫分：三人轮流叫分，首家/后手音效区分，决定地主后发底牌。
- 出牌与要不起：选牌校验牌型，AI 按策略出牌；空牌表示要不起并触发提示与音效。
- 倒计时与超时：用户回合启动倒计时，超时自动出第一张或“不出”。
- 牌型识别与比较：识别顺子、飞机、炸弹、王炸等并支持压制判定。
- 机器人 AI：根据局势选择首出/压牌组合，兼顾队友与出完可能性。
- 结算与继续：按地主/农民倍数结算，播放结果音效，允许继续新局。
- 视听效果：背景、头像、动画及多类音效来自资源与 JSON 播放列表。

## 4. 非功能需求与约束
- 依赖 Qt Widgets 与 Qt Multimedia，C++17；资源通过 `res.qrc` 内嵌。
- 单机本地运行，实时性由 Qt 事件驱动保障。

## 5. 运行环境
- 需要支持 Qt 的桌面环境（Windows/Linux/macOS 均可），具备多媒体解码能力。

## 6. 用户需求
- 可视化手牌展示与拖拽/点击选牌，明确的“出牌/要不起/开始”按钮。
- 明确的角色/要不起提示、倒计时、动画与音效反馈。
- 支持连续多局累加分数并重新开局。
